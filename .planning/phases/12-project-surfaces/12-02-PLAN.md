2. Regenerate model_comparison.csv from the same data. The generate_all_tables function creates .md and .json but NOT .csv. Write a small script to produce it:
   ```python
   from cathode_ml.evaluation.metrics import load_all_results, MODELS_ORDER, PROPERTIES
   import csv
   results = load_all_results()
   with open('data/results/comparison/model_comparison.csv', 'w', newline='') as f:
       w = csv.writer(f)
       w.writerow(['property', 'model', 'mae', 'rmse', 'r2', 'n_train', 'n_test'])
       for prop in PROPERTIES:
           for model in MODELS_ORDER:
               if model in results.get(prop, {}):
                   m = results[prop][model]
                   w.writerow([prop, model.upper() if model in ('rf',) else model.capitalize() if model == 'xgb' else model.upper(),
                               m['mae'], m['rmse'], m['r2'], m['n_train'], m['n_test']])
   ```
   IMPORTANT: Use the MODEL_LABELS dict for consistent model names in CSV: RF, XGBoost, CGCNN, M3GNet+dagger, TensorNet. Actually, looking at existing CSV, model names used are: RF, CGCNN, M3GNet, TensorNet (plain strings). Keep that convention but add XGBoost rows.

   Simpler approach -- just use load_all_results and MODELS_ORDER:
   ```python
   from cathode_ml.evaluation.metrics import load_all_results, MODELS_ORDER, MODEL_LABELS, PROPERTIES
   import csv
   results = load_all_results()
   with open('data/results/comparison/model_comparison.csv', 'w', newline='') as f:
       w = csv.writer(f)
       w.writerow(['property', 'model', 'mae', 'rmse', 'r2', 'n_train', 'n_test'])
       for prop in PROPERTIES:
           for model in MODELS_ORDER:
               if model in results.get(prop, {}):
                   m = results[prop][model]
                   label = MODEL_LABELS[model].replace('\u2020', '')  # strip dagger for CSV
                   w.writerow([prop, label, m['mae'], m['rmse'], m['r2'], m['n_train'], m['n_test']])
   print('CSV written')
   ```
