Create cathode_ml/models/train_megnet.py mirroring the structure of train_cgcnn.py but using matgl's Lightning trainer instead of GNNTrainer. ALL matgl/DGL/Lightning imports must be lazy (inside functions, not at module top level).

Key functions:

1. `_set_seeds(seed: int)` - Same as train_cgcnn._set_seeds (random, numpy, torch seeds).

2. `convert_lightning_logs(log_dir: str, output_csv: str) -> None` - Reads Lightning CSVLogger output (metrics.csv in log_dir), merges train and val rows by epoch (Lightning logs them on separate rows), renames columns to project standard format (epoch, train_loss, val_loss, val_mae, train_mae). Writes to output_csv. Handle column name variations: matgl's ModelLightningModule may log as "train_Total_Loss", "val_Total_Loss", "train_MAE", "val_MAE" -- inspect actual column names and map accordingly. If exact column names are unknown, read the CSV and map any column containing "loss" (case-insensitive) to train_loss/val_loss and any containing "mae" (case-insensitive) to train_mae/val_mae, split by train/val prefix.

3. `train_megnet_for_property(structures, targets, property_name, megnet_config, seed) -> dict` - Core training function for one property:
   a. Lazy import matgl, matgl.ext._pymatgen_dgl (Structure2Graph, get_element_list), matgl.graph._data_dgl (MGLDataLoader, MGLDataset, collate_fn_graph), matgl.utils.training (ModelLightningModule), lightning (L), lightning.pytorch.loggers (CSVLogger), lightning.pytorch.callbacks (ModelCheckpoint, EarlyStopping)
   b. Load pretrained model via load_megnet_model() from megnet.py
   c. Build Structure2Graph converter using pretrained model's cutoff (NOT CGCNN's 8.0A)
   d. Create MGLDataset with structures and targets
   e. Create MGLDataLoader with collate_fn_graph, batch_size from config
   f. Create ModelLightningModule wrapping the pretrained model with LR from config
   g. Configure Lightning Trainer with: CSVLogger to results_dir, ModelCheckpoint monitoring val_MAE (save_top_k=1), EarlyStopping on val_MAE with patience from config, max_epochs from config, accelerator="auto"
   h. Call trainer.fit()
   i. After training: convert Lightning .ckpt checkpoint to .pt using get_megnet_state_dict() and torch.save(). Save as megnet_{property}_best.pt and megnet_{property}_final.pt in results_dir.
   j. Convert Lightning logs via convert_lightning_logs()
   k. Evaluate on test set: use predict_with_megnet() on test structures, then compute_metrics()
   l. Return metrics dict {mae, rmse, r2, n_train, n_test}

4. `train_megnet(records, features_config, megnet_config, seed=42) -> dict` - Main orchestrator (mirrors train_cgcnn):
   a. Load target_properties from features_config
   b. Set seeds
   c. For each property: filter records with valid values, skip if <5, do compositional_split with SAME parameters as CGCNN (same seed, same test_size/val_size from features_config), convert records to pymatgen Structures, extract targets, call train_megnet_for_property()
   d. Collect results as {property: {megnet: metrics}}
   e. Save to results_dir/megnet_results.json via save_results()
   f. Log summary

5. `__main__` block: argparse with --seed flag, loads configs/megnet.yaml and configs/features.yaml via load_config, loads data/processed/materials.json, calls train_megnet(). Same pattern as train_cgcnn __main__.
