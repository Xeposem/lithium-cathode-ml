Create M3GNet and TensorNet training orchestrators and wire them into the pipeline CLI.

Purpose: Enable end-to-end training for both new architectures using matgl 2.x Lightning integration. M3GNet fine-tunes from pretrained (like MEGNet did); TensorNet trains from scratch. Both use identical compositional splits as CGCNN/baselines for fair comparison.

Output: Two training orchestrator modules and an updated pipeline.py that replaces megnet with m3gnet + tensornet in the CLI.
</objective>

```python
from functools import partial
collate_fn = partial(collate_fn_graph, include_line_graph=True)  # M3GNet
train_loader, val_loader, test_loader = MGLDataLoader(
    train_data=train_data, val_data=val_data, test_data=test_data,
    collate_fn=collate_fn, batch_size=128, num_workers=0,
)
```

dataset = MGLDataset(

)

lit_module = ModelLightningModule(model=model, include_line_graph=True, lr=0.001)

Create `cathode_ml/models/train_m3gnet.py` following the structure of `train_megnet.py` but with matgl 2.x API changes:

**Module docstring:** "M3GNet training orchestrator with matgl 2.x Lightning integration." Mention fine-tuning from pretrained weights, compositional group splitting, per-property sequential training.

**Top-level imports (non-lazy):**
- csv, json, logging, random, pathlib.Path
- numpy, torch
- pymatgen.core.Structure
- cathode_ml.data.schemas.MaterialRecord
- cathode_ml.features.split: compositional_split, get_group_keys
- cathode_ml.models.m3gnet: get_m3gnet_state_dict, load_m3gnet_model, predict_with_m3gnet
- cathode_ml.models.utils: compute_metrics, save_results

**`_set_seeds(seed: int)`:** Identical to train_megnet.py.

**`convert_lightning_logs(log_path, output_csv)`:** Identical to train_megnet.py (reuse same function).

**`_run_lightning_training(model, train_structures, train_targets, val_structures, val_targets, property_name, m3gnet_config, seed)`:**
Key differences from train_megnet.py:
1. Lazy imports: `from matgl.ext.pymatgen import Structure2Graph, get_element_list`
2. Lazy imports: `from matgl.graph.data import MGLDataLoader, MGLDataset, collate_fn_graph`
3. Lazy imports: `from matgl.utils.training import ModelLightningModule`
4. Lazy imports: `from functools import partial`
5. MGLDataset must include `threebody_cutoff=4.0` and `include_line_graph=True`:
   ```python
   train_dataset = MGLDataset(
       threebody_cutoff=4.0,
       structures=train_structures,
       converter=converter,
       labels={"Eform": train_targets},
       include_line_graph=True,
   )
   ```
6. collate_fn must be wrapped with partial:
   ```python
   collate_fn = partial(collate_fn_graph, include_line_graph=True)
   ```
7. MGLDataLoader uses v2.x keyword-arg API. Since we have separate train/val datasets (not a single dataset split), create loaders individually:
   ```python
   from torch.utils.data import DataLoader
   train_loader = DataLoader(
       train_dataset, batch_size=batch_size, shuffle=True,
       collate_fn=collate_fn, num_workers=0,
   )
   val_loader = DataLoader(
       val_dataset, batch_size=batch_size, shuffle=False,
       collate_fn=collate_fn, num_workers=0,
   )
   ```
   NOTE: If MGLDataLoader in v2.x only accepts the (train_data, val_data, test_data) keyword API with split_dataset results, use torch DataLoader directly instead. The collate_fn handles batching.
8. ModelLightningModule: `ModelLightningModule(model=model, include_line_graph=True, lr=lr)`
9. Use training config params: `decay_steps` and `decay_alpha` for the built-in CosineAnnealingLR (passed to ModelLightningModule if it accepts them, otherwise just lr).
10. Checkpoint filename: `m3gnet_{property_name}_best`
11. Monitor: `val_MAE`
12. Save .pt checkpoints via `get_m3gnet_state_dict(model)`

**`train_m3gnet_for_property(...)`:** Same structure as train_megnet_for_property but calls load_m3gnet_model and predict_with_m3gnet.

**`train_m3gnet(records, features_config, m3gnet_config, seed=42)`:** Same structure as train_megnet but:
- Results key is `"m3gnet"` not `"megnet"`
- Results file is `m3gnet_results.json`
- Default results_dir is `data/results/m3gnet`
- Logs say "M3GNet" not "MEGNet"

**`if __name__ == "__main__":` block:** Same as train_megnet.py but loads `m3gnet.yaml`.

   )

  3. Then train and evaluate

- Results key is `"tensornet"`
- Results file is `tensornet_results.json`
