For each target property in features_config["target_properties"]: filter to records that have that property (not None), subset X and y, train RF and XGBoost, evaluate both, collect results. Save combined results JSON to baselines_config["results_dir"]/baseline_results.json. Return the results dict.
       - Per project decision (STATE.md): separate models per property, NOT multi-output.

For run_baselines integration test, create a list of fake MaterialRecord objects with known formulas and properties. Mock featurize_compositions to return synthetic features (avoid slow matminer in tests). Verify JSON output file is created and contains expected structure.
